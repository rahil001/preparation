Question: Things that happens in one tick;
1. Node looks for pending timers like are ready to be called (timeout, interval, immediate)
2. Node looks for pending OS operations and calls revelant callbacks (filesystem operations or some crypto libs).
3. Node looks for pending OS Tasks and calls revelant callbacks.
Event Loop is a single threaded. But behind the scene c++ part of Node is comprises of libUV which has 4 threads(thread pool) by default
So Node js offloads the computationally expensive task to libuv (thread pool multithreaded)
Questions Answers
1 Can we use threadpool for javascript code or can only node js fucntions use it ? we can write custom js that uses the thread pool
2 What functions in Node std library uses the threadpool ? All file system and crypto modules stuff. Also depends on OS
3 How does threadpool stuff fit in eventLoop ? TaskRunning in the threadpool are the `pendingOperation` that node js checks during a single tick.
4 What functions in Node std library are handled by OS's asyns features? Almost everything around networking for all Os's. Some other stuff is OS specific
5 How does OS stuff fit in the eventLoop ? Tasks using the underlying OS are reflected in our 'pendingOSTasks' array
